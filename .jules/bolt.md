## 2024-05-24 - [Pandas iterrows Performance]
**Learning:** Iterating through Pandas DataFrames using `iterrows()` is extremely slow because it boxes each row into a Pandas Series object. This adds significant overhead and causes blocking operations, especially inside tight loops like constructing responses for the frontend. Using `ruff --unsafe-fixes` blindly on this codebase often aggressively breaks code by deleting partial variable initializations without checking effects on logic or star-import availability.
**Action:** When working with Pandas DataFrames inside loops, avoid `iterrows()`. Instead, use vectorized operations when possible. If iteration is required to build lists of dictionaries, use `zip()` on specific columns (e.g., `zip(df['col1'], df['col2'])`) or use `df.to_dict('records')`. Always verify Ruff fixes manually rather than applying them globally across the backend.
